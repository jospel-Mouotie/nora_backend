1. PRÉSENTATION GÉNÉRALE
1.1 Concept
MBOA SHOP est une plateforme e-commerce où tous les produits sont obligatoirement rattachés à une boutique validée par l'admin. La plateforme intègre un système de réels (vidéos courtes) permettant aux utilisateurs et boutiques certifiées de publier du contenu. Les vues génèrent des gains en monnaie virtuelle utilisables pour acheter des produits.

1.2 Acteurs
#	Acteur	Rôle
1	Client	Achete, regarde des vidéos, gagne MB Coins, poste 1 vidéo/semaine, peut s'abonner aux boutiques
2	Commerçant	Vends, catalogue gratuit, assigne livreur, poste vidéos, gère ses propres publicités
3	Grossiste	Vends en gros, catalogue gratuit, assigne livreur, poste vidéos, gère ses propres publicités
4	Livreur	Livre, chat client, suivi GPS, scan QR code
5	Admin	Gère tout, valide comptes et boutiques, commission, chat avec clients
1.3 Règles Fondamentales
Règle	Description
Produits	Un produit est TOUJOURS rattaché à une boutique
Boutique	Doit être validée par l'admin pour vendre
Pas de boutique = Pas de produit	Impossible d'ajouter un produit sans boutique active

2.10 Commandes & Livraison
2.10.1 Code PIN
Règle	Description
Génération	Automatique à chaque commande
Format	6 chiffres
Envoi	Notification push au client
Expiration	48 heures
Tentatives	5 maximum
2.10.2 QR Code
Règle	Description
Génération	Automatique à chaque commande
Contenu	ID commande + PIN + timestamp
Utilisation	Scan par le livreur pour validation
Affichage	Client voit son QR code dans l'app
Sécurité	QR code unique, expire avec le PIN
Fonctionnement :

text
1. Livreur arrive chez client
2. Client montre son QR code
3. Livreur scanne le QR code
4. Validation automatique de la livraison
5. Option alternative : saisie manuelle du PIN
2.10.3 Validation Croisée
Validation	Qui	Quand
Client	Donne son QR code ou PIN	Après réception colis
Livreur	Scanne le QR code ou entre PIN	Après livraison
Les deux validations sont REQUISES pour finaliser la commande.

7. FLUX COMPLET
text
1. Client s'inscrit
2. Client peut s'abonner/liker des boutiques
3. Client regarde des vidéos (gagne MB Coins)
4. Client publie une vidéo (1/semaine)
5. Client accumule MB Coins
6. Client achète produit (espèces ou MB Coins)
          ↓
     Commande validée
     → Génération PIN (6 chiffres)
     → Génération QR code unique
          ↓
     Admin/Commerçant assigne livreur
          ↓
     Livreur reçoit mission
          ↓
     Client suit livreur (GPS)
          ↓
     Livreur arrive chez client
     Client montre QR code OU donne PIN
     Livreur scanne OU entre PIN
          ↓
     Validation automatique
          ↓
     Client reçoit notification si oublie
          ↓
     Client note livreur

10. CENTRES D'INTÉRÊT & HABITUDES PERSONNALISÉES
10.1 Onboarding - Sélection des intérêts
Étape	Description
1. Inscription terminée	L'utilisateur arrive sur l'écran de sélection
2. Présentation catégories	Liste des catégories disponibles avec icônes
3. Sélection multiple	L'utilisateur coche ses centres d'intérêt (minimum 3)
4. Niveau d'intérêt	Pour chaque catégorie : 1 (Peu) à 5 (Passionné)
5. Validation	Enregistrement des préférences
6. Accueil personnalisé	Page d'accueil adaptée aux choix

Règles de sélection :
- Minimum 3 catégories obligatoires
- Maximum 10 catégories
- Niveau d'intérêt obligatoire pour chaque catégorie
- Possibilité de modifier plus tard dans les paramètres
